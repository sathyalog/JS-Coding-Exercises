Closures in JavaScript

Closure is nothing but when you have a function defined inside of another function where inner function has access to variables and scope of the outer function even if the outer function finishes executing those variables that no longer accessible outside the function.

Closures 


                function Adder(x) {
                    return function(y){
                      return x + y
                    }
                }
                  
                const add5 = new Adder(5);
                console.log(add5(2)); // 7
                const add10 = new Adder(10);
                console.log(add10(3)); //13
Realtime use case: Closures are very useful when you have some private information which should not be accessible to global scope. A normal function which is a closure can only call the private function and can access the data of it. By this your private function information is not accessible for rest of the functions/global functions. 

                function main() {
                    var privatefunction = function () {
                        console.log('hello');
                    }
                
                    return {
                        publicfunction : function () {
                            privatefunction();
                        }
                    }
                }
                
                const myFunc = new main;
                myFunc.publicfunction();//hello
